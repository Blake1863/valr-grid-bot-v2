## Context Caching — Final Decision

**Tested:** qwen3.5-plus vs qwen3-max with caching

**Findings:**
- qwen3.5-plus: Explicit cache only (not supported by OpenClaw)
- qwen3-max: Implicit cache (requires session continuity — OpenClaw doesn't pass session IDs)
- **Result:** Neither model gets cache benefits in current OpenClaw architecture

**Cost comparison (no cache):**
| Model | Input/1M | Output/1M | 100 req (50k ctx) |
|-------|----------|-----------|-------------------|
| qwen3.5-plus | $0.40 | $2.40 | **$2.24** |
| qwen3-max | $1.20 | $6.00 | $6.60 (3x more) |

**Decision:** Switched back to **qwen3.5-plus** — 67% cheaper without cache benefits.

**Future:** If OpenClaw adds explicit cache support or session continuity, could save 50-70% with caching.
