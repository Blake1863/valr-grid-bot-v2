## CM Bot v2 (Perp) — Issues Fixed

### Root Causes Identified
1. **Race condition** — `tokio::join!` fired maker+taker simultaneously. 5ms sleep before the futures were created, not before they were polled. Taker hit external book before maker landed.
2. **Pre-cycle cleanup bug** — Was cancelling the previous cycle's resting GTC maker before the new taker could match it. Both accounts went long.
3. **Balance check wrong field** — Used `available` (spot field) instead of `totalInReference` (futures margin).
4. **Old binary** — WS was using old subscribe pattern, server closed immediately. Fixed: no subscribe needed, auto-pushes on connect.
5. **`?pair=` filter ignored** — VALR returns all open orders regardless. Fixed: filter client-side.
6. **WS reconnect storm** — 2s backoff caused rate limiting (429). Fixed: 5s+ exponential backoff on errors, immediate reconnect on clean close.

### Final Architecture
- **Order placement**: Account WS `PLACE_LIMIT_ORDER` → await `PLACE_LIMIT_WS_RESPONSE` (~10ms)
- **Sequencing**: Await maker confirmation, then send taker IOC immediately
- **Price**: Mid from `AGGREGATED_ORDERBOOK_UPDATE`, mark fallback if spread >50bps
- **Balance**: `totalInReference` from USDT `BALANCE_UPDATE` event
- **Ping/pong**: Every 20s to keep WS alive

## Key VALR API Lessons
- Account WS: no subscribe needed, auto-pushes on connect
- `PLACE_LIMIT_WS_RESPONSE` is the correct response type (not `ORDER_PLACED`)
- `clientMsgId` correlates order requests to responses
- Spot trade history: `/v1/account/tradehistory?currencyPair=X&limit=N`
- Position field: `pair` not `currencyPair` in `/v1/positions/open`
- Mark price from `MARKET_SUMMARY_UPDATE` WS event (`markPrice` field)
- `?currencyPair=` filter is IGNORED by VALR for both open orders AND trade history — always filter client-side
- Transfer API: `POST /v1/account/subaccounts/transfer` — `fromId`/`toId` (0=main), `currencyCode`, `amount`, `allowBorrow`
- ZAR currency code in balances is "R" not "ZAR" — but transfer uses "ZAR"
- `/v1/account/subaccounts` endpoint must NOT have a subaccount header — causes 401
- Internal transfer amount must be short decimal string (e.g. "16.0" not "16.000000") — too many decimal places causes 403
- BNB price tick size is 0.1 (1dp) not 2dp
