# 2026-04-06 — VALR Grid Bot v2 (TypeScript rewrite + production launch)

## What happened

Full ground-up rewrite of the VALR grid bot. The old Rust bot had fatal bugs
and was replaced with a clean TypeScript implementation built directly against
the VALR API docs.

## Old bot (Rust) — why it was broken

- Used `entryPrice` field (doesn't exist) → stop-loss trigger was always 0
- Hardcoded tick decimal places instead of loading from pair metadata
- Wrong DELETE path for conditional orders
- No startup reconciliation → would double-up orders on restart
- No dry-run mode
- Subaccount auth broken (returned 401)
- `allowMargin: true` on futures orders → rejected with "Margin not enabled" (that flag is for spot margin, not perps)

## New bot (TypeScript) — location

`bots/valr-grid-bot-v2/`

### Key learnings during launch

1. **`allowMargin` is NOT used for futures** — it's a spot margin flag. Remove it from all futures order payloads.
2. **API key is main-account only** — subaccount header causes 401. `subaccountId: ""` in config.
3. **Account WS stale timeout** — account WS only pushes on order events, not continuously. Stale check must be ≥5 min, not 30s.
4. **Batch PLACE_LIMIT supports same fields as single orders** — minus `allowMargin` for futures.
5. **VALR `DELETE /v1/orders/{currencyPair}`** — cancels all open orders for pair, no body needed.
6. **Level parsing on reconcile** — regex `grid-[A-Z]-N-seed` correctly extracts level from all customerOrderId formats.
